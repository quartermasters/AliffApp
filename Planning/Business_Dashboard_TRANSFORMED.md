## 3. SDL Integration: Solicitation Diagnosis Lab

### 3.1 SDL Overview

**SDL (Solicitation Diagnosis Lab)** is a multi-AI orchestration system that breaks solicitation analysis into **34 discrete tasks** across **3 phases**.

### 3.2 SDL Three-Phase Framework

#### **PHASE 1: TRIAGE** (AI-Led, Human-Supervised)
**Duration**: 4-8 hours
**AI Strategy**: Single AI execution, cost-optimized

**Tasks 1-11**:
- Document ingestion and parsing (Claude - long-context strength)
- Metadata extraction (Gemini - data extraction)
- Requirements extraction (Gemini)
- Evaluation criteria mapping (Gemini)
- Compliance requirements (Claude - regulatory focus)
- Timeline/deadline mapping (Gemini)
- Complexity scoring 1-10 (GPT-5 - reasoning required)
- Risk flag detection (GPT-5 - pattern recognition)

**Output**:
- Structured solicitation data
- Complexity score (determines analysis depth: Standard/Complex/Strategic)
- Risk flags identified
- **Human Decision Point**: Senior strategist assigns diagnosis depth level

---

#### **PHASE 2: STRATEGIC INTELLIGENCE** (Human-Led, AI-Assisted)
**Duration**: 2-3 days
**AI Strategy**: Parallel execution for research, Multi-AI consensus for critical analysis

**Research Tasks (12-16) - Parallel Execution**:
- Incumbent research (Gemini - data aggregation)
- Agency spending patterns (Gemini - data analysis)
- Contracting officer background (Grok - real-time information)
- Recent similar awards and outcomes (Gemini - historical data)
- Competitor analysis (Gemini + Grok combined)

**Critical Analysis Tasks (17-22) - Multi-AI Cross-Validation**:

**THESE REQUIRE CONSENSUS FROM MULTIPLE AIs:**

**Task 17: Why is this being re-competed NOW?**
- AIs: GPT-5, Claude, Gemini (all three analyze)
- Process:
  1. Each AI independently analyzes timing and context
  2. System compares outputs for consensus/conflict
  3. If consensus (2+ agree): Document finding with confidence score
  4. If conflict: Escalate to human expert with all three perspectives
- Output: Strategic insight on procurement timing

**Task 18: Unstated Requirement Detection**
- AIs: GPT-5, Claude (cross-validation)
- Process:
  1. Both AIs identify requirements that "should be there but aren't"
  2. Both AIs identify constraints not explicitly stated
  3. Consensus on high-confidence unstated requirements
  4. Conflicts escalated to human
- Output: List of probable unstated requirements (US Embassy security clearance example)
- **Core competitive advantage - requires strategic thinking from multiple angles**

**Task 19: Real Pain Point Identification**
- AIs: GPT-5, Claude (cross-validation)
- Process:
  1. Both AIs analyze what agency really needs vs. what's written
  2. Both AIs identify operational constraints
  3. Consensus synthesis
- Output: Real pain points beyond stated requirements
- **"Fever vs. diagnosis" - critical strategic insight**

**Task 20: Budget Reality Assessment**
- AIs: GPT-5, Gemini (cross-validation)
- Process:
  1. GPT-5 assesses budget reasonableness based on deliverables
  2. Gemini analyzes historical contract values for similar work
  3. Compare findings
- Output: Budget reality (realistic, underestimated, overestimated) + implications

**Task 21: Timeline Constraint Analysis**
- AIs: GPT-5, Claude (cross-validation)
- Process:
  1. Both AIs analyze timeline feasibility
  2. Both AIs identify operational constraints (security clearances, facility access, etc.)
  3. Consensus on critical timeline factors
- Output: Timeline reality assessment + constraint identification

**Task 22: Agency Culture Assessment**
- AI: GPT-5 (strategic analysis)
- Input: Agency history + past awards + solicitation language tone
- Output: Culture profile (Risk-averse vs. innovation-friendly, Relationship-driven vs. process-driven, Quality vs. price focused)

**Pattern Recognition Tasks (23-25)**:
- Match against known patterns library
- Red flag identification (wired for incumbent, unrealistic budget/timeline)
- Green flag identification (incumbent vulnerability, client strengths align)

**Output**:
- Diagnosis brief with unstated drivers
- Real pain points identified
- Competitive landscape mapped
- Win probability estimate (preliminary)
- **Ready for Win Strategy development**

---

#### **PHASE 3: WIN STRATEGY** (Collaborative - Multi-AI + Human)
**Duration**: 1-2 days
**AI Strategy**: Multi-AI for critical decisions, single AI for support tasks

**Strategic Recommendations Tasks (26-29) - Multi-AI Cross-Validation**:

**Task 26: Win Probability Assessment**
- AIs: GPT-5, Claude (consensus required)
- Process:
  1. Both AIs independently calculate win probability
  2. Both provide rationale
  3. If within 10% agreement: Average with confidence score
  4. If >10% divergence: Escalate to human expert
- Output: Win probability % + confidence level + rationale

**Task 27: Win Theme Generation**
- AIs: GPT-5, Claude, Gemini (all three contribute)
- Process:
  1. Each AI generates 3-5 potential win themes
  2. System aggregates unique themes (10-15 total)
  3. Each AI scores themes for impact (1-10)
  4. Top 3-5 themes by consensus score selected
- Output: 3-5 win themes addressing diagnosed pain points

**Task 28: Differentiator Identification**
- AIs: GPT-5, Claude (cross-validation)
- Process:
  1. Both AIs identify client differentiators vs. competitors
  2. Both AIs map differentiators to evaluation criteria
  3. Consensus on strongest differentiators
- Output: Top 3-5 differentiators with supporting evidence

**Task 29: No-Bid Recommendation**
- AIs: GPT-5, Claude (consensus required)
- Process:
  1. Both AIs independently assess bid/no-bid
  2. If both recommend no-bid: Strong recommendation to client
  3. If one recommends no-bid: Present both perspectives to human expert
  4. Human makes final no-bid decision
- Output: Bid/No-Bid recommendation with rationale
- **CRITICAL business decision - requires multi-AI + human validation**

**Solution Design Support Tasks (30-34) - Single AI**:
- Solution approach recommendations (GPT-5)
- Risk mitigation strategies (GPT-5)
- Teaming recommendations (Gemini - research-based)
- Pricing strategy guidance (GPT-5 - strategic)
- Capture plan outline (GPT-5 - strategic planning)

**Output**:
- Strategic brief with win themes
- Differentiators mapped to evaluation criteria
- Recommended approach
- Risk mitigation strategies
- Bid/No-bid recommendation
- Capture plan outline

### 3.3 SDL Consensus Logic

**When Multi-AI Cross-Validation Required**:
1. Strategic insight (Tasks 17-22)
2. Win probability assessment (Task 26)
3. Win theme generation (Task 27)
4. Differentiator identification (Task 28)
5. No-bid recommendation (Task 29)

**Consensus Scenarios**:

**Scenario 1: Full Consensus (All AIs Agree)**
- Action: Document finding with high confidence score (90-95%)
- Human Review: Optional, proceed with finding

**Scenario 2: Majority Consensus (2 out of 3 Agree)**
- Action: Document majority finding with medium confidence score (70-85%)
- Human Review: Recommended, especially if dissenting AI has strong rationale

**Scenario 3: Split Decision (No Consensus)**
- Action: Flag as high-uncertainty case
- Human Review: **MANDATORY** - Expert adjudicates
- Output: Present all AI perspectives to human, document human decision + rationale
- Learning: Often reveals critical insight - investigate why AIs disagreed

**Scenario 4: AI Confidence Scores Conflict**
- Action: If AI expresses low confidence (<70%) even with consensus, escalate
- Human Review: Recommended

### 5.2 Access Control Implementation

```php
// File: /public/includes/classes/PortalAccessControl.php

class PortalAccessControl {

    // Define portal types
    const PORTAL_SUPER_ADMIN = 'super_admin';
    const PORTAL_CLIENT = 'client';
    const PORTAL_TEAM = 'team';

    /**
     * Check if current user can access portal type
     */
    public static function canAccessPortal($portalType) {
        $user = getCurrentUser();

        switch ($portalType) {
            case self::PORTAL_SUPER_ADMIN:
                return $user['type'] === 'admin' && $user['role'] === 'Super Admin';

            case self::PORTAL_CLIENT:
                return $user['type'] === 'client';

            case self::PORTAL_TEAM:
                return $user['type'] === 'team';

            default:
                return false;
        }
    }

    /**
     * Get accessible projects for current user
     */
    public static function getAccessibleProjects() {
        $user = getCurrentUser();
        $db = getDBConnection();

        if ($user['type'] === 'admin' && $user['role'] === 'Super Admin') {
            // Super Admin: see ALL projects
            $stmt = $db->query("SELECT * FROM bd_projects WHERE status = 'active'");
            return $stmt->fetchAll();
        }

        if ($user['type'] === 'client') {
            // Client: see only THEIR projects
            $stmt = $db->prepare("
                SELECT p.* FROM bd_projects p
                INNER JOIN bd_project_clients pc ON p.id = pc.project_id
                WHERE pc.client_id = ? AND p.status = 'active'
            ");
            $stmt->execute([$user['id']]);
            return $stmt->fetchAll();
        }

        if ($user['type'] === 'team') {
            // Team Member: see only ASSIGNED projects (anonymized)
            $stmt = $db->prepare("
                SELECT
                    p.id,
                    p.project_code,
                    p.project_codename AS project_title, -- Use codename, not real client name
                    NULL AS client_name, -- Hide client name
                    p.industry_category,
                    p.current_stage,
                    p.deadline,
                    pa.assignment_type,
                    pa.status AS assignment_status
                FROM bd_projects p
                INNER JOIN bd_project_assignments pa ON p.id = pa.project_id
                WHERE pa.team_member_id = ? AND p.status = 'active'
            ");
            $stmt->execute([$user['id']]);
            return $stmt->fetchAll();
        }

        return [];
    }

    /**
     * Anonymize project data for team view
     */
    public static function anonymizeProjectForTeam($project) {
        return [
            'id' => $project['id'],
            'project_code' => $project['project_code'],
            'project_title' => $project['project_codename'], // NOT real client name
            'industry_category' => $project['industry_category'],
            'contract_type' => $project['contract_type'],
            'value_range' => $project['value_range'], // Range, not exact value
            'current_stage' => $project['current_stage'],
            'deadline' => $project['deadline'],
            // EXCLUDED: client_name, client_contact, client_location, exact_value
        ];
    }
}
```

```php
    /**
     * Execute multi-AI task with consensus logic
     */
    private function executeMultiAITask($task) {
        // Update status
        $this->updateTaskStatus($task['id'], 'processing');

        // Get project documents
        $documents = $this->getProjectDocuments();

        // Call primary AI
        $primaryResult = $this->aiRouter->callAI(
            $task['primary_ai_provider'],
            $documents,
            $task['task_name']
        );

        // Call secondary AI
        $secondaryResult = $this->aiRouter->callAI(
            $task['secondary_ai_provider'],
            $documents,
            $task['task_name']
        );

        // For critical tasks, also call tertiary AI (Gemini)
        $tertiaryResult = null;
        if (in_array($task['task_number'], [17, 18, 19, 27])) {
            $tertiaryResult = $this->aiRouter->callAI(
                'gemini',
                $documents,
                $task['task_name']
            );
        }

        // Build consensus
        $consensus = $this->buildConsensus($primaryResult, $secondaryResult, $tertiaryResult);

        // Save to consensus log
        $stmt = $this->db->prepare("
            INSERT INTO bd_sdl_consensus_log
            (task_id, project_id, task_name, gpt5_output, claude_output, gemini_output, consensus_type, consensus_confidence, final_result, escalated_to_human)
            VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
        ");

        $stmt->execute([
            $task['id'],
            $this->projectId,
            $task['task_name'],
            json_encode($primaryResult),
            json_encode($secondaryResult),
            $tertiaryResult ? json_encode($tertiaryResult) : null,
            $consensus['type'],
            $consensus['confidence'],
            json_encode($consensus['result']),
            $consensus['escalate_to_human']
        ]);

        // If requires human validation or split decision, escalate
        if ($consensus['escalate_to_human'] || $task['requires_human_validation']) {
            $this->updateTaskStatus($task['id'], 'escalated_to_human');
            $this->notifySuperAdmin("Task {$task['task_number']} requires human validation");
        } else {
            // Update task as completed
            $stmt = $this->db->prepare("
                UPDATE bd_sdl_task_queue
                SET status = 'completed',
                    primary_ai_result = ?,
                    secondary_ai_result = ?,
                    consensus_result = ?,
                    confidence_score = ?,
                    completed_at = NOW()
                WHERE id = ?
            ");

            $stmt->execute([
                json_encode($primaryResult),
                json_encode($secondaryResult),
                json_encode($consensus['result']),
                $consensus['confidence'],
                $task['id']
            ]);
        }

        // Log activity
        $this->logActivity("SDL Multi-AI Task {$task['task_number']} completed ({$consensus['type']})");
    }

    /**
     * Build consensus from multiple AI outputs
     */
    private function buildConsensus($primary, $secondary, $tertiary = null) {
        // Compare outputs
        $similarity = $this->calculateSimilarity($primary['text'], $secondary['text']);

        if ($tertiary) {
            $similarity2 = $this->calculateSimilarity($primary['text'], $tertiary['text']);
            $similarity3 = $this->calculateSimilarity($secondary['text'], $tertiary['text']);
            $avgSimilarity = ($similarity + $similarity2 + $similarity3) / 3;
        } else {
            $avgSimilarity = $similarity;
        }

        // Determine consensus type
        if ($avgSimilarity >= 0.8) {
            // Full consensus
            return [
                'type' => 'full_consensus',
                'confidence' => 90 + ($avgSimilarity * 10),
                'result' => $primary, // All agree, use primary
                'escalate_to_human' => false
            ];
        } elseif ($avgSimilarity >= 0.6) {
            // Majority consensus
            return [
                'type' => 'majority_consensus',
                'confidence' => 70 + ($avgSimilarity * 20),
                'result' => $this->synthesizeOutputs([$primary, $secondary, $tertiary]),
                'escalate_to_human' => true // Recommend human review
            ];
        } else {
            // Split decision
            return [
                'type' => 'split_decision',
                'confidence' => 40 + ($avgSimilarity * 30),
                'result' => ['primary' => $primary, 'secondary' => $secondary, 'tertiary' => $tertiary],
                'escalate_to_human' => true // MANDATORY human review
            ];
        }
    }
```
